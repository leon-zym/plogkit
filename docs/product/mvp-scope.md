# MVP 范围

## 总览

第一版包含三个编辑能力，外加导出与最小限度的编辑会话韧性。

三个编辑能力基于**统一文档模型**（见 [ADR 0003](../adr/0003-document-driven-architecture.md)）：它们不是三个互斥模式，而是同一文档上可叠加的正交能力——例如拼接后的长图上可以继续加字。UI 以任务导向的入口流呈现，数据层始终是同一个文档。

各功能的可执行验收标准见 [`docs/specs/`](../specs/)，本文只描述范围边界。

## 1. 为图片加字

必备能力：

- 在图片之上或周围添加文本。
- 支持长段落与合理换行（中文排版为一等公民）。
- 提供一小组干净的文本样式。
- 支持字号、颜色、对齐与行高。
- 在合适场景支持文本背景块。

约束：

- 第一版不提供装饰性字效。
- 不包含贴纸类文本素材。
- 不暴露复杂排版系统。

## 2. 添加背景

必备能力：

- 设置背景色。
- 支持常用输出比例：1:1、3:4、4:5、9:16 及原始比例。

体验要求：这应当是一个快速任务流，而不是图层编辑器。

## 3. 多图拼接

必备能力：

- 竖向拼接。
- 网格拼接。
- 可调图片间距。
- 背景色。

约束：

- 第一版不暴露完全自由布局。
- 常见拼接场景不要求用户手动摆放每张图。

## 4. 导出与压缩预设

必备能力：

- 提供若干面向社交发布场景的导出预设。
- 预设为数据驱动结构（见 [ADR 0008](../adr/0008-export-presets-data-driven.md)），各平台（微信朋友圈、小红书、抖音等）的具体分辨率、画质与格式参数由维护者后续定义，结构先行。
- 支持 JPEG 与 PNG。
- 质量与体积的取舍对用户可理解。

尺寸与语义约束（见 [ADR 0007](../adr/0007-export-pipeline.md)）：

- 输出上限：总像素 ≤ 64MP 且长边 ≤ 16384px，超限自动降级。
- "原始"预设的语义为"在上限内尽量保留"，不承诺无条件保留原尺寸。

元数据（见 [ADR 0008](../adr/0008-export-presets-data-driven.md)）：

- 默认剥离全部 EXIF（含 GPS 定位）。
- App 设置提供全局开关；导出面板提供本次导出的临时开关。

明确的 MVP 限制（见 [ADR 0009](../adr/0009-sdr-export-live-photo-still.md)）：

- 导出统一为 SDR，HDR（增益图）不保留，列 vNext 重评估。
- Live Photo 导入取封面静帧，Live 属性不保留，列 vNext 重评估。
- Display P3 源图经合成后统一导出为 sRGB；MVP 不承诺广色域保真（见 [ADR 0018](../adr/0018-mvp-srgb-color-strategy.md)）。

产品措辞纪律：

- 不承诺预设能阻止社交平台上传后的二次压缩。
- 应用能优化上传前的导出，平台侧处理不在控制范围内。

初始预设方向（占位，待参数定义）：

- 原始：在上限内尽量保留尺寸，高质量。
- 社交：平台友好的尺寸与质量。
- 紧凑：更小体积，便于快速分享。
- 自定义：可后置，非第一版必需。

## 5. 撤销与重做

第一版包含撤销重做。

理由：文字、背景、间距与拼接调整极易误操作；轻量应用可以省略重型项目管理，但基础的失误恢复是感知质量的一部分。

实现原则（见 [ADR 0004](../adr/0004-state-management-undo.md)）：

- 在有意义的提交点记录文档状态变化。
- 手势与滑杆过程不逐帧记录。
- 有界历史栈，30–50 步。

## 6. 当前编辑会话自动保存

第一版包含当前编辑会话的自动保存。

必备行为：

- 从后台返回不丢失当前编辑。
- 意外退出不立即销毁当前项目。
- 导出不压平唯一可编辑副本。

这不等于完整草稿列表。

## 7. 导出后继续编辑

导出后当前项目在会话内保持可编辑。完整导出历史与项目管理可以延后。

## 有意延后的功能

- 完整草稿列表与多项目管理。
- 自由画布入口。
- 图层索引控制与复杂图层 UI。
- 任意图片图层的透明度控制。
- 模板库、品牌套件。
- 云同步、账户系统。
- AI 编辑、视频。
- Share Extension（v1.1，见 [ADR 0017](../adr/0017-share-extension-deferred.md)）。
- HDR 保留与 Live Photo 导出（vNext，见 [ADR 0009](../adr/0009-sdr-export-live-photo-still.md)）。

## MVP 成功标准

第一版成功的标志是用户能够：

- 快速打开应用并选择照片。
- 顺畅地添加可读性良好的文字。
- 几步内完成干净的背景。
- 将多张图拼接为可发布的构图。
- 导出适合社交发布的结果。
- 通过撤销或自动会话恢复从常见失误中恢复。

以上标准的可执行化（Given/When/Then 场景）见 [`docs/specs/`](../specs/)。
