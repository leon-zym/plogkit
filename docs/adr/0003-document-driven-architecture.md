# ADR 0003：文档驱动的编辑器架构与统一文档模型

- 状态：已接受（2026-07-02）
- 关联：ADR 0004、0007、0011

## 背景

编辑状态需要支撑撤销重做、会话自动保存、导出后继续编辑与可预期的导出渲染。同时，MVP 的三个编辑能力（加字、边框背景、拼接）需要明确它们与数据模型的关系：是三个互斥模式，还是同一文档上可叠加的能力（例如给拼接后的长图加字）。

## 决策

- 编辑状态表达为可序列化的文档（document），是唯一事实源。简化结构包含：画布设置、源图引用、文本元素、边框/边距/背景设置、拼接布局设置、导出设置。
- 文档 schema 自带 `schemaVersion` 字段，从第一版开始随附迁移入口；schema 变更必须升版本并提供迁移。
- 采用统一文档模型：三个编辑能力是同一文档上的正交能力，可以叠加（拼接结果上可加字）。UI 以任务导向的入口流呈现，数据层始终是同一个文档。
- 文本元素使用画布绝对坐标，可选锚定到图块；不引入图层索引等复杂图层概念。
- 运行时对象（Skia 图像、手势状态、动画 shared values）不作为持久化事实源。

## 影响与代价

- 文档为纯 JSON、图片按引用存储，体积小，使快照式撤销（ADR 0004）与自动保存成本极低。
- "文档 → Skia 元素树"可实现为纯函数，支撑无头渲染回归测试（ADR 0011）。
- 统一模型要求入口流的 UI 设计保持任务流外观，避免暴露成自由画布；这是设计约束而非数据约束。
