# ADR 0008：导出预设数据驱动与 EXIF 策略

- 状态：已接受（2026-07-02）
- 关联：ADR 0007

## 背景

各社交平台（微信朋友圈、小红书、抖音、Instagram 等）对分辨率、画质、格式的要求不同，各平台的参数定义尚未完成；未来还可能加入平台特定的后处理（如锐化）。导出功能第一版需简单实现，但代码必须预留扩展性。EXIF 处理涉及隐私（GPS 定位），需要明确的产品策略。

## 决策

- 导出预设定义为声明式数据结构，渲染与编码管线只消费预设数据，不硬编码平台逻辑。预设字段至少包含：标识与名称、目标平台、尺寸规则（短边/长边/比例约束）、输出格式、质量参数、后处理链（`postProcess`，预留锐化等）、元数据策略。
- 预设结构自带 `presetSchemaVersion`，与文档 schema 独立演进。
- 具体平台参数后续定义，当前以占位预设起步（原始/社交/紧凑三档方向，见产品文档）；新增平台预设应当只是新增一条数据。
- EXIF 策略：默认剥离全部 EXIF（含 GPS）；App 设置中提供全局开关，导出面板提供本次导出的临时开关；两级开关均可选择保留拍摄时间/设备信息。
- 产品措辞纪律：不承诺预设能阻止平台上传后的二次压缩。

## 影响与代价

- 数据驱动结构略增前期设计成本，换来平台参数定义与代码实现的解耦。
- 预设参数未定不阻塞导出功能开发：管线先行，参数后填。
