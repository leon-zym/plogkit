# 领域文档

工程 skills 探索代码库时，应按以下规则读取本仓库的领域文档。

## 探索前读取

- 根目录的 `CONTEXT.md`。
- 若根目录存在 `CONTEXT-MAP.md`，则按其指引读取与当前主题有关的各个 `CONTEXT.md`。
- 阅读 `docs/adr/` 中与即将处理的区域相关的 ADR。

文件不存在时静默继续，不提前建议创建。`/domain-modeling` 会在术语或决策真正明确时按需创建这些文件。

## 文件结构

本仓库采用 single-context：

```text
/
├── CONTEXT.md
├── docs/adr/
└── src/
```

## 使用词汇表中的术语

Issue 标题、重构建议、假设和测试名称等内容，应使用 `CONTEXT.md` 定义的领域术语，避免采用词汇表明确排除的同义词。

所需概念尚未出现在词汇表中时，应先判断是否引入了项目未使用的语言；若确属领域模型缺口，则记录并交由 `/domain-modeling` 处理。

## 标明与 ADR 的冲突

若输出内容与现有 ADR 冲突，应明确指出，不得静默覆盖既有决策。
