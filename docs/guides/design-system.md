# MVP 界面设计方向

本文记录 MVP 的界面视觉与交互基线。它服务于实现一致性，不改变 `docs/product/` 与
`docs/specs/` 的功能边界。

## 方向锁定

- **视觉主张**：安静、温暖、直接的照片工作台。控制界面像浅暖纸张，编辑画布置于石墨色舞台，照片始终是第一视觉锚点。
- **内容计划**：首页只负责选择照片与恢复当前会话；编辑器负责预览、撤销重做、任务工具与导出；工具面板按背景、拼接、文字、导出逐项展开。
- **交互主张**：按钮按压轻微缩放，工具面板短距离上移淡入，选择变化使用低幅度交叉淡入；不使用弹跳、装饰性转场和持续动画。

实现样式只使用 React Native `StyleSheet` 与集中 token，不引入第二套样式系统。

## 参考产品与取舍

- [Apple Photos](https://support.apple.com/guide/iphone/edit-photos-and-videos-iphb08064d57/ios)：照片占据主区域，完成/取消等会话动作放在顶部，具体编辑工具和精确调节放在底部。PlogKit 采用同样的内容主导和上下分区，但不复制滤镜与调色能力。
- [Adobe Express](https://www.adobe.com/express/feature/image/editor)：从图片直接进入编辑，再按当前任务逐步暴露工具。PlogKit 采用渐进式工具入口，但不引入模板、素材库、动画和 AI。
- [Canva](https://www.canva.com/features/)：比例、拼接与文字入口易发现。PlogKit 保留这种任务命名的直白性，但拒绝自由画布、庞大工具目录和账号/云工作流。

## 1. 视觉主题与氛围

首页为暖色浅底，强调轻量和日记感；编辑器主画布区使用深石墨底，让不同明度的照片都能看清边缘。控制面板回到暖色浅底，避免全屏暗色带来的压迫感。界面不使用渐变、玻璃模糊和装饰性卡片网格。

视觉签名是画布或选中元素周围的朱砂色短角标。它表达当前编辑对象，不用厚边框遮挡照片。

## 2. 色彩与角色

以下 hex 值由感知均匀的暖中性色与朱砂强调方向换算而来，React Native 代码使用 hex 以保证跨端支持。

| Token | 值 | 角色 |
|---|---|---|
| `canvasWarm` | `#F6F1E8` | 首页与控制面板主背景 |
| `surface` | `#FFFCF7` | 输入区、工具区和浮起操作面 |
| `stage` | `#242321` | 照片编辑舞台 |
| `ink` | `#1D1B18` | 浅色面主文字 |
| `inkMuted` | `#716B62` | 浅色面辅助文字 |
| `line` | `#DED7CC` | 分隔线与不可交互轮廓 |
| `accent` | `#D95D3F` | 选择、主操作和导出确认 |
| `accentPressed` | `#BB472F` | 主操作按下态 |
| `accentSoft` | `#F4D8CF` | 选择背景，不承载灰色文字 |
| `stageText` | `#F4F0EA` | 深色舞台文字与控件 |
| `stageMuted` | `#B9B2A8` | 深色舞台辅助文字 |
| `danger` | `#B93A32` | 删除与不可恢复错误 |
| `success` | `#347A55` | 导出与保存成功 |

遵循 60-30-10：大面积为暖中性背景，照片和石墨舞台构成次级视觉重量，朱砂只用于当前选择和关键确认。

## 3. 排印规则

品牌关键词为“安静、触感、直接”。拒绝将 Inter、DM Sans、Space Grotesk 作为无意识的展示字体。MVP 的应用 UI 有意使用平台系统字体：iOS 使用 SF Pro/PingFang SC，Android 使用 Roboto/Noto Sans CJK fallback，以保证中文、动态字号、输入法和跨端可读性。照片内文字由文档中的逻辑字体 ID 映射，和 UI 字体分离。

| 层级 | 字号 | 字重 | 行高 | 用途 |
|---|---:|---:|---:|---|
| Display | 34 | 700 | 40 | 首页产品名 |
| Title | 22 | 650 | 28 | 页面与工具标题 |
| Body | 16 | 400 | 25（中文）/ 23（英文） | 说明和输入 |
| Label | 14 | 600 | 19 | 按钮与工具标签 |
| Caption | 12 | 500 | 17 | 状态、尺寸和辅助信息 |

32pt 以上英文标题可使用轻微负字距，中文不使用负字距。动态数字使用等宽数字特性（平台支持时启用）。

## 4. 组件样式

- 主按钮：朱砂底、浅色文字、12pt 圆角、最小 48pt 高；按下时缩放至 0.97 并切换 `accentPressed`；禁用时保留形状，透明度 0.38。
- 次按钮：透明底、`ink` 文字；按下时使用 `accentSoft`，不添加边框卡片。
- 图标按钮：可见区域至少 40×40pt，实际点击区域至少 44×44pt；必须有 `testID` 与 `accessibilityLabel`。
- 工具导航：编辑器底部固定一行文字+简洁符号，当前项以朱砂短下划线和字重表达，不用填满整格的胶囊背景。
- 工具面板：`surface` 背景，从底部内联展开；顶部仅一条细分隔线，不使用模糊玻璃。
- 输入：`surface` 上的 1px `line` 轮廓，聚焦时变为朱砂；错误信息紧邻输入，不能只用颜色表示。
- 色板：圆形色样仅用于颜色；选中态使用外圈和勾选语义，点击区域不小于 44pt。
- 图片缩略条：保持原图比例的缩略图，可用上下/左右按钮调整顺序；选中项显示短角标，不使用重阴影。
- 完成反馈：直接显示“已保存到相册”及尺寸/格式，细节可展开；不显示无内容的统计项。

圆角 token 固定为 `r4=4`、`r12=12`、`r20=20`、`pill=999`。只有色样、状态点和极短的单选标签使用 pill。

## 5. 布局原则

- 间距 token：`4, 8, 12, 16, 24, 32, 48`。
- 首页左右边距 24pt，主要内容自顶部安全区下方开始，不做垂直完全居中的模板式欢迎页。
- 编辑器按“顶部会话栏 / 中央舞台 / 缩略条 / 底部工具与面板”分区。照片舞台优先获得剩余高度。
- 面板内同类控件使用 12pt 间距，面板内边距与控件间距默认同为 16pt，避免逐项微调。
- 不用卡片承载普通文本。仅对需要独立点击和状态识别的“恢复当前会话”使用一个抬升面。

## 6. 深度与层级

浅色相邻面通过至少 4% 明度差区分。只定义两级阴影：

- `level1`：`0 1 3 rgba(29,27,24,0.12)`，用于恢复会话与轻量浮起操作。
- `level2`：`0 8 24 rgba(29,27,24,0.16)`，仅用于需要聚焦的文本输入面板或导出确认面板。

深色舞台不使用黑色投影，靠 `stage` 与控制面的明度差建立层级。图片使用 1px 内描边以维持边界，不改变布局尺寸。

## 7. 应做与不应做

- 应让照片成为首页选图后和编辑器中的最大视觉对象。
- 应让每个工具面板只完成一个任务，状态变化立即反馈到预览。
- 应让中文长文、英文长标签和 320pt 宽屏幕都可完整操作。
- 应用文字说明 SDR/sRGB、EXIF 与尺寸降级等真实限制。
- 不使用紫蓝渐变、玻璃模糊、三列同构卡片和无意义装饰插画。
- 不提供自由图层列表、模板市场或“更多 AI 工具”等越界入口。
- 不用隐藏的滑动手势承载唯一操作路径，核心动作必须有可见控件。
- 不因界面简洁而隐藏撤销、恢复和导出隐私选项。

## 8. 响应式与无障碍

- MVP 手机宽度范围按 320 到 430pt 验证，iPad 暂不支持；横屏不作为主流程。
- 所有触控目标至少 44×44pt，系统动态字号开启后工具标签允许两行，不能尾部省略关键动作。
- 使用 Safe Area；键盘出现时文本输入与确认按钮保持可达。
- 所有交互元素具备 `testID`、本地化 `accessibilityLabel`、角色、选中/禁用状态。
- 动效仅改变 `transform` 与 `opacity`，时长 120 到 180ms；系统减少动态效果开启时取消位移，仅保留必要淡入。

## 9. Agent 实现速查

颜色：`canvasWarm #F6F1E8`、`surface #FFFCF7`、`stage #242321`、`ink #1D1B18`、`inkMuted #716B62`、`line #DED7CC`、`accent #D95D3F`、`accentPressed #BB472F`、`accentSoft #F4D8CF`。

- 首页：在 `#F6F1E8` 上使用 24pt 水平边距；PlogKit 为 34pt/700；主选图按钮 48pt 高、`#D95D3F`、12pt 圆角；不要居中 hero 或卡片网格。
- 编辑器：顶部栏 52pt，舞台 `#242321` 填满剩余空间；底部工具标签 14pt/600；当前工具用 2pt 朱砂短下划线；面板 `#FFFCF7`、16pt 内边距。
- 工具面板：标题 22pt/650，正文 16pt/25；控件间距 12pt；色样 32pt 可见直径配 44pt 点击区；禁用态透明度 0.38。
- 文本输入：底部内联面板使用 `level2` 阴影，输入轮廓 `#DED7CC`，聚焦 `#D95D3F`；确认按钮 48pt 高；键盘出现后仍可见。
- 导出完成：首行只显示“已保存到相册”，16pt/600、`#347A55`；下一行 12pt 显示格式与尺寸；没有内容的详情行不渲染。
