# ADR 0014：语言策略：中文权威文档 + 英文代码与提交

- 状态：已接受（2026-07-02）
- 关联：ADR 0013、0015

## 背景

项目将在 GitHub 开源，需兼顾国际协作。中文为当前文档的主要语言。全量双语同步维护存在文档漂移风险。

## 决策

- 单一权威源原则：每类文档只维护一份权威版本。
  - `docs/`（product、adr、specs、guides）：以中文为权威版本。
  - `README.md`（英文）与 `README.zh-Hans.md`（中文）：双语维护，属唯一例外（面向 GitHub 访客、低频改动）。
  - 代码注释、commit message、`AGENTS.md`：英文。
- 应用 UI：中英双语，i18n 层自第一天建立，无硬编码文案。
- `docs/` 的英文翻译将择机完成，届时评估是否切换权威语言（如切换则新增 ADR）。

## 影响与代价

- 开源初期国际贡献者阅读 `docs/` 需借助翻译，属有意识的取舍。
- README 双语存在低频漂移风险，修改任一份时必须同步另一份（AGENTS.md 中约束）。
