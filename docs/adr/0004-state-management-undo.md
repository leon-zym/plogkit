# ADR 0004：状态管理与撤销重做：Zustand + 有界快照栈

- 状态：已接受（2026-07-02）
- 关联：ADR 0003

## 背景

文档驱动架构需要一个承载文档状态的运行时容器，并在其上实现撤销重做。候选包括 Redux Toolkit、Zustand、React context + useReducer，撤销实现候选包括全量快照与补丁（如 immer patches）。

## 决策

- 文档状态存放于 Zustand store；组件按需订阅，避免全量重渲染。
- 撤销重做采用有界快照栈：在有意义的提交点保存整份文档 JSON，栈深 30–50。
- 不采用补丁式撤销（immer patches 等）：文档体积小（图片按引用），快照成本可忽略，补丁属过早优化。
- 手势与滑杆等高频中间态保留在 Reanimated shared values 中，仅在手势结束/控件释放时提交进文档，不逐帧记录。
- 撤销提交点与自动保存共用同一事件流：每次提交即触发自动保存调度。

## 影响与代价

- 实现简单、可测试性强（core 纯逻辑不依赖 React）。
- 若未来文档体积显著增大（如内嵌大量矢量数据），再评估切换为补丁式，届时新增 ADR 取代本决策。
