# ADR 0015：许可证：GPL-3.0 + CLA，资产许可纪律

- 状态：已接受（2026-07-02）

## 背景

项目的诉求：允许开发者协作，但禁止闭源套壳盈利；同时保留未来闭源上架的可能性。GPL 与 App Store 分发存在著名的许可冲突（VLC 案例），这对第三方套壳者构成双重阻拦；而版权人不受自己许可证约束，可对自有代码重新授权——前提是保有全部版权或取得贡献者授权。

## 决策

- 代码许可证采用 GPL-3.0-only，仓库根目录放置 LICENSE 全文。
- 自开始接受外部贡献起启用 CLA（贡献者许可协议，将重授权权利让渡给维护者），开源上线时配置 CLA bot。
- 资产许可纪律（与代码许可无关，独立执行）：项目引入的字体、图标等资产一律只选择允许商业闭源嵌入的许可——字体用 OFL（如霞鹜文楷、思源系），图标与其他资产用 MIT/Apache-2.0/CC-BY。禁止引入 GPL/AGPL/CC-BY-NC 等会限制未来闭源分发的资产。

## 影响与代价

- GPL 的强 copyleft 会劝退部分潜在商业使用者，符合项目意图。
- 未来若闭源上架，需确认全部外部贡献均在 CLA 覆盖下，否则需剥离或重写相应代码。
