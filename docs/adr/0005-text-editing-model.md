# ADR 0005：文本编辑模型：原生输入提交 + Skia Paragraph 渲染

- 状态：已接受（2026-07-02）
- 关联：ADR 0001、0003

## 背景

Skia 自绘文本输入无法妥善处理光标、选区、键盘与中文 IME；而原生排版与 Skia Paragraph 排版存在不可消除的细微差异，试图把原生输入框像素级叠加在 Skia 渲染文字上做内联 WYSIWYG 会陷入对齐泥潭。

## 决策

- 文本的活跃编辑使用原生输入组件，以面板/底部输入区形式呈现，不做内联叠加 WYSIWYG。
- 编辑提交后写入文档模型，由 Skia Paragraph API 渲染最终效果；用户在提交后立即看到最终渲染结果。
- 中文与多语言渲染依赖 Paragraph 的系统字体 fallback（iOS 上为 PingFang 系）。
- 文档模型存储逻辑字体 ID，由各平台/测试环境的字体注册表映射到具体字体；随包附带 1–2 款 OFL 授权的风格字体（候选：霞鹜文楷）。
- MVP 不提供装饰性字效、贴纸文本与复杂排版系统；支持字号、颜色、对齐、行高与文本背景块。

## 影响与代价

- 输入体验交给系统，规避 IME 与选区的全部复杂度；代价是编辑态与最终渲染态存在视觉切换。
- Skia Paragraph 的 emoji 行高问题（上游 issue #3422）需要在实现时验证，必要时对含 emoji 文本做行高修正或在 MVP 文档中声明限制。
- 逻辑字体 ID 的间接层是无头渲染测试字体确定性的前提（ADR 0011）。
